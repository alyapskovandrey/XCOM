<b>Работа с Тикетами</b>

<b>Тикет</b> - это отчет о каждом проделанном действии или обработанном обращении. Например: проконсультировал абонента по оплате; позвонил абоненту, но не дозвонился — создай <b>тикет</b>.

<b>Для чего нужны тикеты?</b>

<div>1. Для расчета зарплаты.</div><div><br /></div><div>2. Для просмотра истории и причин обращений клиента/абонента.</div><div><br /></div><div>3. Для аналитики и оптимизации процессов на основании полученных данных.</div>

<b>Когда нужно создавать тикеты?</b>

<div>1. Любое обращение</div><div><br /></div><div align="left">2. <b>Обработка периодических задач, таких как например: </b><br /></div><div><br /></div><div> * Проверка оплат.</div><div><br /></div><div> * Обзвон</div><div><br /></div><div> * Прием и отправка посылок.</div><div><br /></div><div> * Выдача, контроль документов.</div><div><br /></div><div> * и тд.</div>

<b>Когда не нужно создавать тикеты?</b>

<div>Тикет можно создать 2-мя способами: <br /></div><div><br /></div><div>Можно создать новый тикет заполнив поля формы вручную</div>

<div align="left">Но удобнее создавать тикет с RS. В открытом меняю абонента, в разделе "Техподдержка".</div>

При создании тикета с RS некоторые поля заполняются автоматически.

<b> Тикет состоит из нескольких блоков: </b>

<div>*     Контактные номера абонента. Можно выбрать несколько или один номер из списка с которого обращался абонент или на который звонил оператор.</div><div><br /></div><div>*     Адрес абонента.</div><div><br /></div><div>*     Информация об абоненте. Здесь RS ID абонента, номер договора, автор создавший этот тикет, название текущего тарифа, количество дней на этом тарифе и название коммутатора, к которому абонент привязан.</div><div><br /></div><div>*     Источник общения. “ИСТОЧНИК” можно выбрать из списка нажав на стрелку справа, самые популярные источники выведены ниже.</div><div><br /></div><div>*     В “ПРИЧИНЫ” указываем почему обратился к нам абонент и создан тикет, в “РЕШЕНИЕ” чем оператор помог абоненту. Если причин и решений несколько, фиксируем каждую причину и каждое решение.</div><div><br /></div><div>*     Если есть какая-то дополнительная информация, она дописывается в блоке “КОММЕНТАРИЙ”.</div><div><br /></div><div>*     После заполнения всех нужных полей, сохраняем тикет.</div>

<div>* Полный список "Причин" и "Решений" можно посмотреть в Метрике.</div><div><br /></div><div>* В списке черным цветом написано само действие, а серым цветом синонимы ( при написании синонима, появится подходящее действие)</div>

*     Можно зайти в историю тиктов и нарядов, через RS. В открытом меняю абонента, в разделе "Техподдержка".

*     Жмем на название нужного тикета.

<div>*     Нажимаем на “Изменить”, редактируем тикет и сохраняем.</div><div><br /></div><div>* <b><font color="#f44336"> Таким способом можно редактировать тикет в течении 2 дней после создания тикета. </font></b></div>

* Второй способ, через список “Моих тикетов”. Открываем Метрику, жмем на “XCOM МЕТРИКА” => находим нужный тикет в списке и редактируем.

<b>Создание Перезвона</b>

<div><b>Если :</b></div><div><b><br /></b></div><div><b>* </b>Абоненту нужна помощь специалиста, который сейчас не доступен или</div><div><br /></div><div>* Абонент просил перезвонить ему позже,</div><div><br /></div><div>То создайте тикет с причиной "Попросил перезвонить". В списке необработанных обращений такая запись подсветится голубым и будет напоминать все операторам о необходимости ее обработать. Обязательно укажите номер телефона, адрес, дополнительные причины/рпешения, суть вопроса и другие сведения необходимые для повторного контакта с абонентом/клиентом. <br /></div><div><br /></div><div><b>Держите жту страницу открытой всегда! <br /></b></div><div><b><br /></b></div><div><b>Обрабатывайте записи незамедлительно по мере их поступления. <br /></b></div><div><b><br /></b></div><div><b><br /></b></div><div><b> <font color="#f44336">Через этот список, тикеты можно редактировать до конца текущего месяца. </font></b></div>
